Portafolio Backend en Python
  Este repositorio será la base de tu portafolio backend, pensado para crecer paso a paso mientras
  aprendes y construyes proyectos reales.
  La idea no es “llenarlo rápido”, sino que cada proyecto demuestre criterio técnico, orden y progreso,
  algo clave para perfiles backend/junior.
Objetivo del portafolio
  Mostrar tu evolución aprendiendo backend con Python
  Tener proyectos claros, explicables y bien documentados
  Que cualquier recruiter pueda entender:
  Qué problema resolviste
  Qué tecnologías usaste
  Qué aprendiste
Tecnologías objetivo (irán apareciendo de a poco)
  Python
  FastAPI
  SQL (SQLite → PostgreSQL)
  Git / GitHub
  Testing básico
  Autenticación
  Deploy (más adelante)
Nada de esto se hace todo junto. Vamos por capas.

Reglas del portafolio (muy importante)
  Cada proyecto debe tener su propio README.md
  Nada copiado sin entenderlo
  Código simple > código “inteligente”
  Todo proyecto responde a estas 3 preguntas:
    ¿Qué hace?
    ¿Por qué existe?
    ¿Qué aprendí haciéndolo?
